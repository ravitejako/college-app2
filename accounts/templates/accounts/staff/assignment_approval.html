Due Date: Mar 25, 2025

Max Points: 100

Status: Pending

Assignment Description

In this assignment, students will design and create Entity-Relationship (ER) diagrams for a sample database system. The assignment focuses on applying database design principles and normalization techniques.

Learning Objectives
  • Understand and apply database design concepts
  • Create proper ER diagrams using standard notation
  • Identify entities, attributes, and relationships
  • Apply normalization techniques to optimize database structure
Requirements
  1. Students must create an ER diagram for a university registration system
  2. The diagram should include at least 5 entities with appropriate attributes
  3. Identify and properly represent relationships between entities
  4. Include cardinality and participation constraints
  5. Submit a written explanation of design decisions
